## Core Principles (Apply to All Summaries)

**READ THESE CAREFULLY** — These principles are universal regardless of style:

1. **Lead with the Conclusion (Pyramid Principle)**
   - Your FIRST sentence must contain the main finding, answer, or conclusion
   - Never make readers wait for the payoff
   - Structure: Conclusion → Supporting points → Details

2. **Maintain Objectivity**
   - Separate facts from interpretation (label interpretations clearly: "Analysis:", "Interpretation:")
   - Use neutral verbs: "states", "explains", "demonstrates" (not "claims", "admits")
   - Present multiple perspectives when sources disagree
   - Avoid charged adjectives and superlatives

3. **Be Ruthlessly Concise**
   - Every sentence must pass the "so what?" test (does it add value?)
   - Eliminate filler language: "The video discusses..." → just state the information
   - Use concrete data over vague language: "40% faster" not "significantly faster"
   - Aim for maximum information density

4. **Make It Stick (SUCCESs Framework)**
   - **Simple**: Extract the core idea first
   - **Unexpected**: Flag surprising or counterintuitive findings
   - **Concrete**: Use specific numbers, examples, measurements
   - **Credible**: Cite sources (which video)
   - **Emotional**: (when appropriate) Connect to practical impact
   - **Stories**: Use sparingly for complex concepts

5. **Prioritize Actionability**
   - Help readers make decisions or take action
   - Include tradeoffs and caveats (not just benefits)
   - Provide "when to use" / "when not to use" guidance

## Steps

### Step 1: Identify Repeated Logic (3 minutes)

Look for this pattern repeated across components:

```jsx
// ❌ Repeated in every component
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  fetch(url)
    .then(r => r.json())
    .then(setData)
    .catch(setError)
    .finally(() => setLoading(false));
}, [url]);
```

If you see this pattern 2+ times, extract to a custom hook.

---

### Step 2: Extract to Function (5 minutes)

Move the logic to a function that returns what components need:

```javascript
function useData(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // Reset when URL changes
    setLoading(true);
    setError(null);
    
    fetch(url)
      .then(r => {
        if (!r.ok) throw new Error(`HTTP ${r.status}`);
        return r.json();
      })
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}
```

**Key points**:
- Name starts with `[prefix]` ([Technology X] requirement)
- Returns object (easier to extend than array)
- Resets state when URL changes (prevents stale data)

---

### Step 3: Use in Components (1 minute)

Replace repeated logic with one line:

```jsx
function UserProfile() {
  const { data: user, loading, error } = useData('/api/user');

  if (error) return <Error message={error.message} />;
  return <Profile user={user} />;
}

function Posts() {
  const { data: posts, loading, error } = useData('/api/posts');
  // Same pattern, different data
}
```

### Step 4: Add Caching (Optional, 10 minutes)

Prevent re-fetching the same URL:

```javascript
// Outside component (survives re-renders)
const cache = new Map();

function useData(url) {
  const [data, setData] = useState(cache.get(url) || null);
  const [loading, setLoading] = useState(!cache.has(url));
  const [error, setError] = useState(null);

  useEffect(() => {
    if (cache.has(url)) return; // Already cached
    
    setLoading(true);
    fetch(url)
      .then(r => r.json())
      .then(result => {
        cache.set(url, result); // Cache it
        setData(result);
      })
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}
```

**Impact**: First load of `/api/user` fetches, subsequent uses across all components are instant (cached).

---

### Step 5: Add Refresh Capability (5 minutes)

Let components manually refresh data:

```javascript
function useData(url) {
  const [data, setData] = useState(cache.get(url) || null);
  const [loading, setLoading] = useState(!cache.has(url));
  const [error, setError] = useState(null);
  
  // Wrap fetch in useCallback so it can be returned
  const refresh = useCallback(() => {
    setLoading(true);
    setError(null);
    cache.delete(url); // Clear cache
    
    fetch(url)
      .then(r => r.json())
      .then(result => {
        cache.set(url, result);
        setData(result);
      })
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);
  
  // Auto-fetch on mount
  useEffect(() => {
    if (!cache.has(url)) refresh();
  }, [url, refresh]);

  return { data, loading, error, refresh };
}

// Usage
function UserProfile() {
  const { data: user, loading, error, refresh } = useData('/api/user');
  
  return (
    <>
      <Profile user={user} />
      <button onClick={refresh}>Refresh</button>
    </>
  );
}
```

## Visual Mental Model

Think of custom hooks as a vending machine:

```
Component: [Inserts URL] → [useData Machine] → [Returns {data, loading, error}]
                                 ↓
                              [Cache]
                              [Fetch Logic]
                              [Error Handling]
```

The component doesn't care HOW data is fetched—it just provides input (URL) and gets output (data). The hook encapsulates all complexity.

**Layered view**:
```
Layer 1 (Component):     <Profile user={user} />
Layer 2 (Hook):          useData('/api/user')
Layer 3 (Implementation): fetch + state + cache
```

Each layer hides complexity from the layer above.

## Common Pitfalls

**Gotcha 1: Not starting with "use"**
- ❌ `function fetchData(url)` — React won't enforce hook rules
- ✅ `function useData(url)` — React enforces hook rules (no conditional calling, etc.)

**Gotcha 2: Returning arrays vs objects**
- Arrays: `const [data, loading] = useData(url)` — Order matters, can't skip items
- Objects: `const { data, loading } = useData(url)` — Can destructure only what you need
- **Recommendation**: Use objects for hooks with 3+ return values

**Gotcha 3: Not resetting state on input change**
```javascript
// ❌ Bad: Shows stale data while fetching new URL
useEffect(() => {
  fetch(url).then(r => r.json()).then(setData);
}, [url]);

// ✅ Good: Clears old data immediately
useEffect(() => {
  setData(null);        // Reset
  setLoading(true);     // Show loading
  fetch(url).then(r => r.json()).then(setData);
}, [url]);
```

**Surprising: Custom hooks can call custom hooks**
```javascript
function useUser() {
  return useData('/api/user'); // Calls another custom hook
}

function useAuth() {
  const { data: user } = useUser(); // Calls custom hook that calls custom hook
  return user?.isAuthenticated;
}
```

This is valid! Hooks compose like Lego blocks.

## Common Pitfalls

- **Fix**: Check `tailwind.config.js` paths include your components

**Surprising: Custom CSS breaks Tailwind patterns**
If you write custom CSS with `@apply`, you lose Tailwind's benefits. Stay in utility classes 95% of the time.
